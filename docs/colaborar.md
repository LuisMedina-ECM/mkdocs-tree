# 🤝 Cómo colaborar en el manual de TREE

¡Gracias por querer ayudarnos a mejorar! Este proyecto es de código abierto y nos encanta recibir aportaciones de la comunidad. No importa si es corregir una coma o escribir un tutorial entero, **toda ayuda es bienvenida**.

## 📝 ¿Cómo propongo un cambio?

El proceso es muy sencillo y no necesitas saber programar:

1.  **Localiza el error:** Navega hasta la página que quieras mejorar.
2.  **Usa el botón de edición:** Haz clic en el icono del lápiz (ubicado en la parte superior derecha). Esto te llevará al archivo fuente en GitHub.
3.  **Realiza el cambio:** GitHub te permitirá editar el texto directamente en el navegador.
4.  **Envía una "Pull Request":** Al guardar, GitHub te pedirá que describas qué has cambiado. Pulsa en "Propose changes" y luego en "Create Pull Request".

## 📏 Normas de Estilo

Para que el manual mantenga una coherencia, intenta seguir estas pautas:

* **Tono amigable:** Escribe de forma clara y cercana, evitando tecnicismos innecesarios.
* **Formato Markdown:** Usa negritas para destacar elementos de la interfaz (ej: "Haz clic en **Guardar**").
* **Imágenes:** Si añades imágenes, guárdalas en la carpeta `docs/assets/` y asegúrate de que no pesen demasiado.
* **Cuadros de aviso:** Usa los avisos de MkDocs para destacar info importante:

    ```markdown
    !!! info "Información"
        Usa esto para datos relevantes.

    !!! warning "Cuidado"
        Usa esto para evitar que el usuario cometa un error grave.
    ```
